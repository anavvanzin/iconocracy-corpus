## Achados — CRITICAL
- [C-01] **Outputs armazenados nos notebooks 01–04**: células de código contêm imagens PNG embutidas em base64 e stdout armazenado. Versionar outputs em `.ipynb` é anti-pattern (incha repo, dificulta diff, pode romper em merge). Recomendação: executar `jupyter nbconvert --clear-output --inplace` nos 4 notebooks.
- [C-02] **Dependencies ausentes no environment.yml e requirements.txt**: `prince` (notebook 04), `scikit-posthocs` (notebook 02), `seaborn` (todos notebooks 01–08), `statsmodels` (notebook 03), `scipy` (todos). O environment.yml só lista `matplotlib`, `numpy`, `jupyter`, `psycopg2`, `pylint`, `jsonschema`, `rich`. Nenhum desses packages críticos está presente.
- [C-03] **Python kernel mismatch**: environment.yml especifica `python=3.11`. Os notebooks 01–04 declaram kernel `Python 3` com `version: 3.13.12`. Os notebooks 05–08 declaram `Python 3 (ipykernel)` com `version: 3.12.0`. O ambiente conda `iconocracy` usado localmente (segundo AGENTS.md) é Python 3.12 — não 3.11 como no environment.yml, nem 3.13 como nos notebooks 01–04 executados.
