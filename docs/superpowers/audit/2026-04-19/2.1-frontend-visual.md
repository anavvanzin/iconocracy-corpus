## Achados — CRITICAL

- **[C-01] Estática do ensaio visual não reflete estado atual do corpus**
  - Arquivo: `visual-essay-iconocracia.html` (linha 277: `145 artefatos`)
  - Wave 1.1 identificou `companion-data.json` com 145 vs 165 itens (stale)
  - O ensaio visual hardcodes `145 alegorias`, `10 indicadores`, figures com dados específicos — arquivo .html estático, não regenerado desde a atualização do corpus
  - Impacto: leitor sees dados defasados (composite 53.7%, ρ = −0.01/0.27, n=32 nos anos 1910s) sem indication de quando foi gerado
  - Ação: regenerar o HTML automaticamente ou adicionar timestamp/data de geração visível

- **[C-02] `as any` TypeScript não resolvido em 12localizações no iurisvision**
  - Arquivos afetados: `ThesisDashboard.tsx` (linhas 129, 566), `ConnectionMap.tsx` (linhas 142, 160–163), `CorpusExplorer.tsx` (linha 51), `KnowledgeBase.tsx` (linha 396), `firebase.ts` (linhas 12, 18)
  - Padrões: cast de `window` para acessar `aistudio`, cast de `querySnapshot.docs.map` para Firebase, cast de `drag(simulation)` para D3
  - Impacto: erros de runtime não pegos pelo compilador, refactoring perigoso
  - Ação: definir interfaces tipadas parawindow.aistudio, Firebase snapshot types, D3 simulation type

- **[C-03] Charts em canvas/SVG sem acessibilidade de teclado**
  - `visual-essay-iconocracia.html`: Chart.js canvas (`<canvas id='chart-composite'>`) sem tabindex, sem aria-label, role='img' ausente
  - `atlas-iconometrico.html`: Recharts RadarChart (SVG) em #atlas-root, interactivity apenas por onClick em ImageCard, não há tab navigation entre items
  - WCAG 2.1 AA 4.1.2: usuário de teclado não consegue navegar/interagir com dados visualizados
  - Ação: envolver charts em container com role='img' + aria-label descritivo, adicionar navegação por teclado

## Achados — MAJOR

- **[M-01] CDN blocking: Chart.js e React+Recharts bloqueiam render**
  - `visual-essay-iconocracia.html` linha 7: `<script src='https://cdn.jsdelivr.net/npm/chart.js@4.4.0/dist/chart.umd.min.js'>` — síncrono, bloqueia parse
  - `atlas-iconometrico.html` linhas 19–22: React + ReactDOM + Babel standalone + Recharts — todos síncronos do unpkg.com, nenhum defer/async
  - Impacto: First Contentful Paint degradado, fallback offline inexistente
  - Ação: usar `defer` em scripts não-críticos, auto-importar via CDN com module fallback, ou bundlar localmente

- **[M-02] Google Fonts síncrono em todas superfícies**
  - `visual-essay-iconocracia.html` linha 8–9: `<link rel='preconnect'>` + stylesheet síncrono
  - `atlas-iconometrico.html` linhas 10–12: idem
  - `iurisvision/src/index.css` linha 1: `@import url('https://fonts.googleapis.com/...')` — imports bloqueantes
  - Impacto: flash of unstyled text (FOUT), render blocking
  - Ação: usar `font-display: swap`, carregar via `<link rel='preload'>` + `onload` para não-bloqueante

- **[M-03] Companion bundle único de 131KB sem code splitting**
  - Arquivo: `iconocracia-companion/src/app/main.jsx` (2422 linhas, 131KB)
  - Esbuild bundling sem entry points separados, todo o app em um arquivo
  - Impacto: usuário baixa código de features (radiografia, atlas, editorial search) que pode nunca usar
  - Ação: criar chunks separados por rota/feature para lazy loading

- **[M-04] Framing inconsistente entre superfícies — tese vs. simulação penal**
  - `visual-essay-iconocracia.html`: ✅ título correto — Iconografia Jurídica, regimes (fundacional/normativo/militar), ENDURECIMENTO, 10 indicadores
  - `atlas-iconometrico.html`: ⚠️ header diz `Ius Gentium | UFSC · DIREITO INTERNACIONAL` (linha 104) — domínio institucional de direito internacional, não tese de iconografia jurídica
  - `iurisvision/src/App.tsx`: ❌ branding `IURIS VISIO` + gamificação (XP, `awardXP()`), `DefenseSimulator.tsx`, `AIAssistant`, cores de simulação penal (verde exército `#4a7c59`)
  - Impacto: visitante entende que é ferramenta de advocacia criminal, não pesquisa de iconografia jurídica histórica
  - Ação: renomear app para Iconoclass/Iconometria, remover gamificação, usar paleta do ensaio visual (tons quentes acadêmicos)

- **[M-05] Atlas hardcode de apenas 8 itens no CORPUS array**
  - Arquivo: `atlas-iconometrico.html` linha 187–196: array `CORPUS` com 8 itens (BR-001, BR-002, BR-005, BR-006, FR-001, FR-003, UK-003, US-001)
  - ThesisDashboard.tsx usa dados reais do Firestore, Atlas HTML é estático e não conecta ao corpus real
  - Impacto: Atlas visual é demo de UI, não ferramenta de análise real
  - Ação: fazer Atlas buscar de `/api/corpus/data` ou Firestore

## Achados — MINOR

- **[m-01] Hamburguer sem ARIA expand/controls no Atlas**
  - `atlas-iconometrico.html` linha 122: `<button class='hamburger' aria-label='Abrir menu' aria-expanded='false'>` — faltam `aria-controls` e toggle de `aria-expanded`

- **[m-02] Mobile nav duplica semântica nav**
  - `atlas-iconometrico.html` linha 131: `<div class='mobile-nav' role='navigation' aria-label='Navegação mobile'>` — `<nav>` existente com mesma função (linha 107)

- **[m-03] Não há skip link no ensaio visual**
  - `visual-essay-iconocracia.html`: `<header>`, `<main>`, `<footer>` semânticos presentes, mas sem `<a href='#main' class='skip-link'>` para navegação por teclado

- **[m-04] Breakpoint único (640px) no ensaio visual**
  - CSS linha 265: `@media (max-width: 640px)` cobre apenas mobile phones, não há breakpoint para tablets (768px) ou desktops (1440px)
  - `atlas-iconometrico.html`: apenas 768px breakpoint (linha 81), melhor mas ainda incompleto

- **[m-05] CSS sem custom properties compartilhadas**
  - Cada superfície define suas cores inline ou em `:root` isolado — sem design tokens centrais
  - `visual-essay`: `--bg: #faf8f5`, `--accent: #8b1a1a`
  - `atlas`: hardcoded colors em REGIMES object (`#3d1f28`, `#1f2d3d`)
  - `iurisvision`: Tailwind arbitrary values `[#e9dcc9]`, `[#8b5a2b]`, sem central tokenization

- **[m-06] Companion index.js usa CORS permissivo**
  - `iconocracia-companion/src/index.js` linha 8: `'Access-Control-Allow-Origin': '*'` em todas as APIs
  - Funciona para desenvolvimento, mas não é correto para produção com autenticação Cloudflare Access

- **[m-07] `createdAt?: any` em PomodoroSession type**
  - `ThesisDashboard.tsx` linha 32: `createdAt?: any` — tipo incerto para campo Firestore timestamp

- **[m-08] `onError` handler em img sem fallback de accessibility**
  - `atlas-iconometrico.html` linha 259: `onError={e => { e.currentTarget.style.display = 'none'; ... }}` — omite imagem sem alt update, screen reader sees nothing

## Pontos fortes (o que NÃO mexer)

1. **Estrutura semântica HTML do ensaio visual** — uso correto de `<header>`, `<main>`, `<section>`, `<hr>`, `<footer>`, títulos hierárquicos (h1/h2/h3). Esta é a melhor superfície em termos de HTML semântico.

2. **Paleta de cores regime-consistente no ensaio visual** — cores fundacional (#4C72B0), normativo (#DD8452), militar (#C44E52), contra (#55A868) usadas consistentemente em charts e bars. Excelente para comparabilidade visual.

3. **TypeScript estrito no companion** — companion usa JavaScript simples (sem TypeScript), mas o código é limpo, bem-estruturado com funções pequenas e descritivas, sem `any` implícito.

4. **Lazy loading de imagens no Atlas** — `<img loading='lazy'>` usado em ImageCard. Boa prática para performance.

5. **Cache headers bem configurados no companion Worker** — `/vendor/` e `/fonts/` com `immutable`, app.js com `must-revalidate`. Excelente para CDN caching strategy.

6. **Modularização de componentes iurisvision** — cada componente (Dashboard, ThesisDashboard, KnowledgeBase, etc.) em arquivo separado, importações limpas, estrutura de projeto consistente.

7. **Skip link presente no Atlas** — `<a href='#main' class='skip-link'>` é boa prática WCAG, só precisa de styling (não vi `.skip-link` CSS definido).
