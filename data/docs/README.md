# Data Documentation

## Directory Layout

```
data/
├── raw/          # Manifests and Google Drive links only (never raw files)
├── interim/      # Data in transformation (intermediate outputs)
├── processed/    # Datasets ready for analysis
└── docs/         # This documentation
```

## Traceability Rule

Every data item must be traceable across three points:

1. **Origin** → Google Drive (source archive / folder)
2. **Process** → GitHub (script that generated the derived artifact)
3. **Description** → Notion (analytical record, methodological decision)

## Datasets

### `data/processed/feminist_network_48C51_pt.json`

- **Description:** Feminist iconography subnetwork extracted from Iconclass notation 48C51, Portuguese language
- **Size:** ~14 KB
- **Generated by:** `tools/scripts/extract_feminist_network.py`
- **Format:** JSON — hierarchical tree of Iconclass notations with labels and children
- **License:** Derived from Iconclass data

### `corpus/corpus-data.json`

- **Description:** Main research corpus — 66 catalogued iconographic items
- **Size:** ~260 KB
- **Fields:** `id`, `title`, `date`, `period`, `creator`, `institution`, `source_archive`, `country`, `medium`, `motif`, `description`, `url`, `thumbnail_url`, `rights`, `citation_abnt`, `citation_chicago`, `tags`
- **Coverage:** France (13), Germany (11), Brazil (10), USA (9), Portugal (5), Belgium (4), Italy (4), UK (4), Netherlands (3), Austria (2), Switzerland (1)
- **License:** CC BY 4.0

### `examples/batch_001/`

- **Description:** Pilot batch output demonstrating the dual-agent corpus builder pipeline
- **Contents:** batch manifest, WebScout input/output, IconoCode output, master record, JSONL export, ABNT citations
- **Generated by:** `tools/scripts/batch_example.py`
- **See:** `docs/dual-agent-corpus-builder.md` for pipeline specification
